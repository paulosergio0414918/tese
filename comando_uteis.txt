*** class 9.1: Splat and double splat. ***
-> that is an important class. Here we understand the *(star)operand.

[1]print(*a) -> if a is tuple this comand will unpacks a into a sequence to
 print without commas and brackets
[2] zip(a,b) -> is a function that combines multiple iterables 
(lists, tuples, strings, etc.) element-wise into tuples. It creates an 
iterator that aggregates elements from each of the iterables.

Example:
[in] for pair in zip(a, b):
        print(pair)
[out](a_1, b_1)
     (a_2, b_2)
     and so on ...

Example: using star operator
[in] for pair in zip(a, b):
        print(*pair)
[out] a_1, b_1
      a_2, b_2
     and so on ...

Example: it could be in other way
[in] for a_, b_ in zip(a, b):
        print(a_, b_)
[out] a_1, b_1
      a_2, b_2
     and so on ...

-> doble splat does for keyword argument what single splat does for positional 
-> while splat operator packs and umpacks sequences duble splat packs and umpacks dictionaries

[3]**kwargs -> is a special syntax in Python that allows a function
 to accept any number of keyword arguments. It collects these arguments
 into a dictionary where the keys are the argument names (as strings)
 and the values are the argument values.

[4]*args -> is a special syntax that allows a function to accept any number
 of positional arguments. It collects these arguments into a tuple.

Comandos básicos do git
**********************************************************************************
para iniciar o git com em um projeto qualquer devemos seguir os seguintas passos:
echo "# SWE" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin url do repositorio no git exemplo: https://github.com/paulosergio0414918/SWE.git
git push -u origin main

git init -> vai criar uma pasta no arquivo que deseja se enviado ao git
git status -> vai fornecer o status do arquivos presentes na pasta a ser enviada ao git
git add . -> adiciona as mudanças realisadas ao projeto que será eviada ao git
git commit -a -m 'mensagem' -> salva os arquivos e alterações no git

git stash clear -> limpa todas as stash da lista
git stash drop numero_da_stash -> limpa a stash mensionada

git show -> mostra todas as tag existentes no projeto

git remote -v -> nos fornece a origem do repositório
git diff -> é um comando desinado a mostrar a disferença entre a branch que estou com a master

**************************************************
usamos o # para criar títulos, podendo usar de 1 a 6 dependendo da hierarquia
Para colocar um texto em negrito usamos **texto** ou __texto__
para escrever em itálico usamos *texto* ou _texto_
Para construir um lista não ordenada usamos *item
para construir uma lista ordenada usamos 1. item
Também bodemos construir lista aninhadas
1. item
    1. subitem(é preciso colocar 4 espaços na frente do subitem)
    2. subitem
para inserirmos imagem em markdown usamos o comando ![texto alt](link da imagem)
para inserirmos um link no markdown usamos o comando [texto do link](link)
É possível combinar os dois comando acima e colocar um link em uma imagem
para representar um código no git usamos 
''' linguagem
    código
    '''

 - [x] tarefa executada
 - [ ] tarefa não executada
